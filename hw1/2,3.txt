В python 2:
raw_input() - возвращает строку
input() - оценивает ввод и может вернуть строку, число, bool, в зависимости от того что введено. Рассматривает все входные данные как код Python и пытается его выполнить

В python 3: 
raw_input() переименован в input()
input() отменен. Вместо него теперь добавлено eval().  eval(input()) в Python 3, тоже самое что input() в Python 2.


PEP - https://www.python.org/dev/peps/pep-3105/
print - единственная функциональность уровня приложения, для которой есть выделенное утверждение
В какой-то момент в разработке приложений часто возникает необходимость заменить вывод на печать чем-то более сложным, таким как запись вызовов или вызовов в какую-то другую библиотеку ввода / вывода.
Наличие специального синтаксиса для печати создает гораздо больший барьер для эволюции
Нет простого способа преобразовать операторы print в другой вызов, если нужен другой разделитель, без пробелов или вообще без них. Кроме того, не существует простого способа для удобной печати объектов с каким-либо другим разделителем, кроме пробела.
Если print () - это функция, было бы намного проще заменить ее в одном модуле (просто def print (* args): ...) или даже во всей программе (например, поместив другую функцию в __builtin __. Print).
